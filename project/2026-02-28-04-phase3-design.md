## Phase 3 Implementation Plan: Modifiers, ZStack & Remaining Primitives

### Architecture Overview

Every modifier is a `PrimitiveView` struct wrapping `content: any View`. Each intercepts `sizeThatFits` and `render` to transform proposals, responses, or post-process rendered cells. The fluent API lives in a single `View+Modifiers.swift` extension, where each method returns `some View` (which is the concrete modifier type, opaque to the caller).

The key insight is that modifiers fall into three categories:
1. **Layout modifiers** -- transform the size proposal going down and/or the size response coming up: `PaddedView`, `FrameView`, `FlexFrameView`, `FixedSizeView`, `PrioritizedView`, `HiddenView`
2. **Visual modifiers** -- do not affect sizing; post-process rendered cells or fill behind/on top: `BackgroundView`, `OverlayView`, `StyledView`
3. **Structural modifiers** -- affect the shape of the view tree: `BorderedView` (layout + visual)

/// A view that clamps its child's size within min/max bounds.
///
/// Follows SwiftUI's asymmetric clamping rules:
/// - Both min and max set: clamp proposal, propose to child, clamp response
/// - Only min set: propose as-is, max(response, min)
/// - Only max set: min(proposal, max), propose to child, response as-is
struct FlexFrameView: PrimitiveView, Sendable {
    let content: any View
    let minWidth: Int?
    let maxWidth: Int?
    let minHeight: Int?
    let maxHeight: Int?
    let alignment: Alignment

    func sizeThatFits(_ proposal: SizeProposal) -> Size2D {
        let adjustedW = clampProposal(proposal.width, min: minWidth, max: maxWidth)
        let adjustedH = clampProposal(proposal.height, min: minHeight, max: maxHeight)
        let childProposal = SizeProposal(width: adjustedW, height: adjustedH)
        let childSize = TextUI.sizeThatFits(content, proposal: childProposal)
        return Size2D(
            width: clampResponse(childSize.width, min: minWidth, max: maxWidth, proposalHadBoth: minWidth != nil && maxWidth != nil),
            height: clampResponse(childSize.height, min: minHeight, max: maxHeight, proposalHadBoth: minHeight != nil && maxHeight != nil)
        )
    }

    // ... private clamp helpers
}

**FlexFrame clamping logic** (per axis, spelled out):

```swift
private func clampProposal(_ proposal: Int?, min: Int?, max: Int?) -> Int? {
    guard let p = proposal else { return nil }
    switch (min, max) {
    case let (lo?, hi?): return Swift.min(Swift.max(p, lo), hi)  // clamp both
    case (_, nil):       return p                                  // only min: pass through
    case (nil, let hi?): return Swift.min(p, hi)                  // only max: cap proposal
    case (nil, nil):     return p                                  // neither: pass through
    }
}

private func clampResponse(_ response: Int, min: Int?, max: Int?, proposalHadBoth: Bool) -> Int {
    if proposalHadBoth, let lo = min, let hi = max {
        return Swift.min(Swift.max(response, lo), hi)
    }
    if let lo = min { return Swift.max(response, lo) }
    // Only max: response as-is (already capped at proposal stage)
    return response
}
```

/// A view that post-processes the rendered cells to apply style attributes.
///
/// The style is merged additively: `.bold()` adds bold without clearing
/// existing fg color. `.foregroundColor(.red)` sets fg without touching
/// bg or other attributes.
struct StyledView: PrimitiveView, Sendable {

    let styleOverride: Style

    func sizeThatFits(_ proposal: SizeProposal) -> Size2D {
        TextUI.sizeThatFits(content, proposal: proposal)

    func render(into buffer: inout Buffer, region: Region) {
        TextUI.render(content, into: &buffer, region: region)

        for r in region.row ..< min(region.row + region.height, buffer.height) {
            for c in region.col ..< min(region.col + region.width, buffer.width) {
                guard r >= 0, c >= 0 else { continue }
                guard !buffer[r, c].isContinuation else { continue }
                buffer[r, c].style = buffer[r, c].style.merging(styleOverride)

        }

**Important design decision for BackgroundView**: The plan says "render child, then set bg color on cells where bg is nil." But this has a problem: the child's Text renders with `Style.plain` (bg = nil), so the background would overwrite it. Instead, we should fill the background FIRST, then render the child on top. The child's write operations will overwrite cells with their own styles. However, this means the child text cells will have bg = nil while the surrounding padding cells will have the background color. 

Actually, the better approach: fill background first across the entire region, then render content on top. The content's `buffer.write()` will set cells with whatever style the content has. If the content has `bg: nil`, those cells will lose the background. So we need a different approach:

**Revised BackgroundView**: Fill all cells with the background color first. Then render the child. Then post-process: for any cell in the region where bg is nil, set it to the background color. This way, explicitly-set backgrounds from children are preserved, but default-bg cells get the background fill.

Wait, simpler: just fill the entire region with bg color first, then render the child on top. Child cells that have an explicit bg will overwrite. Child cells that have bg=nil... will also overwrite with nil. That is the problem.

The cleanest approach: render the child first, then iterate and set bg on all cells where bg is still nil.

```swift
func render(into buffer: inout Buffer, region: Region) {
    TextUI.render(content, into: &buffer, region: region)
    // Set background on cells that don't have an explicit bg
    for r in region.row ..< min(region.row + region.height, buffer.height) {
        for c in region.col ..< min(region.col + region.width, buffer.width) {
            guard r >= 0, c >= 0 else { continue }
            if buffer[r, c].style.bg == nil {
                buffer[r, c].style.bg = color
            }
        }
    }
}
```

This is the correct approach -- render child, then backfill bg on cells where bg is nil.

/// A view that draws a box-drawing border around its content.
///
/// The border adds 2 columns (left + right edges) and 2 rows
/// (top + bottom edges) to the child's size.
struct BorderedView: PrimitiveView, Sendable {
    let content: any View
    let borderStyle: BorderStyle

    /// The style of box-drawing characters used for the border.
    enum BorderStyle: Friendly {
        /// Rounded corners: ╭╮╰╯ with ─│ edges.
        case rounded
        /// Square corners: ┌┐└┘ with ─│ edges.
        case square

        var topLeft: Character { ... }
        var topRight: Character { ... }
        var bottomLeft: Character { ... }
        var bottomRight: Character { ... }
        var horizontal: Character { "─" }
        var vertical: Character { "│" }
    }

    func sizeThatFits(_ proposal: SizeProposal) -> Size2D {
        let inner = proposal.inset(horizontal: 2, vertical: 2)
        let childSize = TextUI.sizeThatFits(content, proposal: inner)
        return Size2D(
            width: childSize.width + 2,
            height: childSize.height + 2
        )
    }

    func render(into buffer: inout Buffer, region: Region) {
        guard region.width >= 2, region.height >= 2 else { return }
        // Draw corners
        buffer[region.row, region.col] = Cell(char: borderStyle.topLeft)
        buffer[region.row, region.col + region.width - 1] = Cell(char: borderStyle.topRight)
        buffer[region.row + region.height - 1, region.col] = Cell(char: borderStyle.bottomLeft)
        buffer[region.row + region.height - 1, region.col + region.width - 1] = Cell(char: borderStyle.bottomRight)
        // Draw horizontal edges
        buffer.horizontalLine(row: region.row, col: region.col + 1, length: region.width - 2, char: borderStyle.horizontal)
        buffer.horizontalLine(row: region.row + region.height - 1, col: region.col + 1, length: region.width - 2, char: borderStyle.horizontal)
        // Draw vertical edges
        buffer.verticalLine(row: region.row + 1, col: region.col, length: region.height - 2, char: borderStyle.vertical)
        buffer.verticalLine(row: region.row + 1, col: region.col + region.width - 1, length: region.height - 2, char: borderStyle.vertical)
        // Render content inside
        let innerRegion = region.inset(top: 1, left: 1, bottom: 1, right: 1)
        TextUI.render(content, into: &buffer, region: innerRegion)

**StackLayout modification for priority**:

The `layoutGreedy` method needs to be updated. Currently it sorts by flexibility ascending. With priority:

1. Extract priority: if a child is `PrioritizedView`, use its `priority`; otherwise default to `0`.
2. Sort children first by priority descending, then by flexibility ascending within same priority.
3. Allocate in this order: highest-priority, least-flexible children first.

This is a minimal change to the existing sort comparator:

```swift
// In layoutGreedy, change the sort:
flexChildren.sort { a, b in
    let aPriority = (a.view as? PrioritizedView)?.priority ?? 0
    let bPriority = (b.view as? PrioritizedView)?.priority ?? 0
    if aPriority != bPriority {
        return aPriority > bPriority  // higher priority first
    }
    return a.flexibility < b.flexibility  // then least flexible first
}
```

```swift
// Sources/TextUI/Views/ZStack.swift
/// A view that overlays its children back-to-front.
///
/// All children receive the same size proposal. The ZStack's size
/// is the maximum width and height of its children.
///
/// ```swift
/// ZStack {
///     Color(.blue)
///     Text("Overlay")
/// }
/// ```
public struct ZStack: PrimitiveView, Sendable {
    let alignment: Alignment
    let children: [any View]

    public init(
        alignment: Alignment = .center,
        @ViewBuilder content: () -> ViewGroup
    ) {
        self.alignment = alignment
        children = StackLayout.flattenChildren(content().children)
    }

    public func sizeThatFits(_ proposal: SizeProposal) -> Size2D {
        var maxW = 0
        var maxH = 0
        for child in children {
            let size = TextUI.sizeThatFits(child, proposal: proposal)
            maxW = max(maxW, size.width)
            maxH = max(maxH, size.height)
        }
        return Size2D(width: maxW, height: maxH)
    }

    public func render(into buffer: inout Buffer, region: Region) {
        guard !region.isEmpty else { return }
        let stackSize = Size2D(width: region.width, height: region.height)
        for child in children {
            let childSize = TextUI.sizeThatFits(child, proposal: SizeProposal(width: region.width, height: region.height))

// Sources/TextUI/Views/ForEach.swift
/// A view that creates views from a collection of data.
///
/// `ForEach` is layout-transparent: in a stack, its children are
/// flattened into the stack's children rather than being treated
/// as a single view.
///
/// ```swift
/// VStack {
///     ForEach(items) { item in
///         Text(item.name)
///     }
/// }
/// ```
public struct ForEach<Data: RandomAccessCollection & Sendable>: PrimitiveView, LayoutTransparent, Sendable
    where Data.Element: Sendable
{
    let children: [any View]

    public init(_ data: Data, @ViewBuilder content: (Data.Element) -> ViewGroup) {
        children = data.flatMap { element in
            content(element).children
        }
    }

    // LayoutTransparent
    var layoutChildren: [any View] { children }

    // PrimitiveView (for bare usage outside a stack)
    public func sizeThatFits(_ proposal: SizeProposal) -> Size2D {
        guard let first = children.first else { return .zero }
        return TextUI.sizeThatFits(first, proposal: proposal)
    }

    public func render(into buffer: inout Buffer, region: Region) {
        guard let first = children.first else { return }
        TextUI.render(first, into: &buffer, region: region)
    }
}
```

**Important**: The `ForEach` init calls the closure eagerly for each element and stores the resulting views. This is Sendable-safe because the closure is not stored -- only its output is. The `@ViewBuilder content` parameter returns `ViewGroup`, matching the existing stack pattern. However, there is a subtlety: each iteration may produce multiple views (if the closure body has multiple expressions), so we `flatMap` through `content(element).children`.

For the `Identifiable` variant (common in SwiftUI), we can add an overload:

```swift
public init(_ data: Data, @ViewBuilder content: (Data.Element) -> ViewGroup)
    where Data.Element: Identifiable
{
    children = data.flatMap { element in
        content(element).children
    }
}
```

Actually, since we are not doing diffing/reconciliation yet (that is Phase 4 state management territory), the `Identifiable` constraint is not needed. The non-`Identifiable` init is sufficient for now.

// Sources/TextUI/Views/AttributedText.swift
/// A single-line text view with mixed styling spans.
///
/// Unlike ``Text``, which applies one style to the entire string,
/// `AttributedText` supports different styles for different parts
/// of the text.
///
/// ```swift
/// AttributedText {
///     TextSpan("Error: ", style: .bold.foreground(.red))
///     TextSpan("file not found", style: .plain)
/// }
/// ```
public struct AttributedText: PrimitiveView, Sendable {
    let spans: [TextSpan]

    /// A segment of text with a specific style.
    public struct TextSpan: Friendly {
        public let text: String
        public let style: Style

        public init(_ text: String, style: Style = .plain) {
            self.text = text
            self.style = style
        }
    }

    public init(_ spans: [TextSpan]) {
        self.spans = spans
    }

    // Convenience: result-builder-like init
    public init(@SpanBuilder content: () -> [TextSpan]) {
        self.spans = content()
    }

    private var totalWidth: Int {
        spans.reduce(0) { $0 + $1.text.displayWidth }
    }

    public func sizeThatFits(_ proposal: SizeProposal) -> Size2D {
        let w: Int = switch proposal.width {
        case nil: totalWidth
        case 0: 0
        case let proposed?: min(totalWidth, proposed)
        }
        return Size2D(width: w, height: w > 0 ? 1 : 0)
    }

    public func render(into buffer: inout Buffer, region: Region) {
        guard !region.isEmpty else { return }
        var col = region.col
        for span in spans {
            guard col < region.col + region.width else { break }
            let maxLen = region.col + region.width - col
            let text = String(span.text.prefix(maxLen))
            buffer.write(text, row: region.row, col: col, style: span.style)
            col += span.text.displayWidth
        }
    }
}
```

Actually, this is getting complex. Let me look at this from SwiftUI's actual behavior:

- `.frame(minWidth: a, maxWidth: b)`: Frame's width = clamp(proposed, a, b). Child gets proposed the frame's width. Child renders inside frame aligned by `alignment`.
- `.frame(maxWidth: .infinity)`: Frame's width = clamp(proposed, 0, .infinity) = proposed. So frame fills container.
- `.frame(minWidth: 50)`: Frame's width = max(childWidth, 50). Actually no -- Frame's width = max(proposed, 50)? Or max(childResponse, 50)?

SwiftUI documentation says: The view's resulting width is the max of `minWidth` and the child's width, capped at `maxWidth`. But when `maxWidth` is set, it is `min(proposal, maxWidth)` as the frame width... unless the child is bigger.

This is genuinely complex. For the implementation plan, I will document the behavior as follows:

**FlexFrame rule**: The frame's size on each axis is determined by:
1. If only `maxWidth` set: frame width = `min(proposed ?? .max, maxWidth)`. This means the frame takes the proposed width (up to max).
2. If only `minWidth` set: frame width = `max(childWidth, minWidth)`. Child is proposed as-is.
3. If both set: frame width = `clamp(proposed ?? childIdeal, minWidth, maxWidth)`.
4. If neither set: frame width = child width (pass through).

The child is always proposed the frame's resolved size, and renders aligned within it.

This will be refined during implementation with TDD -- write the test cases first based on the definition-of-done (`frame(maxWidth: .max)` makes a hugging view fill), and then implement until they pass.
